Audit or add bilingual localization for this repository.

Steps:

1. Identify hardcoded user-facing strings or the requested translation keys.
2. Add or update keys in both `src/locales/en.json` and `src/locales/ar.json`.
3. Use `$t()` in templates or `t()` from `useI18n()` in scripts.
4. Keep key names module-scoped and consistent between English and Arabic.
5. Check RTL-sensitive UI changes when Arabic layout may be affected.
6. Run `npm run type-check` or a focused validation when localization types are involved.
